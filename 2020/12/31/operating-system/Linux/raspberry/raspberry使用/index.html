raspberry 的使用记录

SSH登录

树莓派使用内存卡写入系统后，如果没有显示屏，需要使用SSH 登录。
把内存卡接到系统上，在读取出的盘上面新建一个名为 ssh 的空文件（没有后缀名）。开机后就可以使用 SSH 登录了。IP地址可以通过IP-Scanner 扫面出来。

树莓派更换镜像源

安装apache2-dev 显示无法连接debina镜像源，需要换源
vi /etc/apt/source.list

vi /etc/apt/sources.list.d/risp.list

然后报错认证错误

树莓派配置文件修改错误导致无法登录

将树莓派的内存卡插入 Linux 发行版系统后，可以识别树莓派的系统文件，这时可以使用发行版的root权限修改树莓派系统配置。但是建议修改树莓派系统时还是先备份修改的文件，之后再重启服务测试，不要直接重启系统。

Deb软件包管理

扩展存储

项使用u盘扩展存储，但是插入U盘后没有反应。使用fdisk -l查看磁盘。
然后确认磁盘节点名字之后，使用mount 命令手动挂载
mount /dev/sda1 /mnt

使用的时之前制作树莓派的内存卡出现inode使用完了的状况。
拷贝文件，虽然还有很大空间，但是显示 No space left on device使用df -ih 查看我呢见节点使用率100%。然后重新格式化内存卡mkfs -t ext4 -c /dev/sda

格式化存储

使用mkfs命令。
mkfs -t ext4 -c /dev/sda
mkfs.ext4 /dev/sda
