多数元素问题

给定一个大小为n的数组，找到其中的多数元素。多数元素是指在数组中出现次数大于[n/2]的元素。
你可以假设数组是非空的，并且给定的数组总是存在多数元素。
示例：
输入 [3，2，3]
输出 3

略

2、哈希计数，复杂度On

python3中有一个collections类其中的Counter()方法，可以统计List作为参数的元素和元素数量组成的dict。

复杂度分析:

  • 时间复杂度：数组长度是n。遍历一次数组中的每一个元素，插入哈希表的时间是常数时间。结束后在对哈希表遍历选最大值。
    这样就是O(n)的复杂度。
  • 空间复杂度
    哈希表最多有n-(n/2)个键值对，所以占用的空间为O(n).

排序法

思路

数组中有众数，那么排序后，无论从大到小还是从小到大，下标为[n/2]的元素必须为众数。

复杂度分析

  • 时间复杂度：排序的时间复杂度 O(nlog n)
  • 空间复杂度：O(log n),排序算法的空间复杂度。

随机化

思路

因为超过[n/2]的数组元素为众数，这样随机选一个下标验证即可。

算法

复杂度分析

  • 时间复杂度：最大为O()
  • 空间复杂度： O(1)

分治

算法

使用经典的分治算法递归求解，直到所有的子问题都是长度为1的数组。长度为1的子数组中唯一的数显然是众数，直接返回即可。
如果回溯后某区间的长度大于1，我们必须将左右子区间的值合并。如果他们的众数相同，那么显然这一段区间的众数是他们相同的值。
否则我们要比较两个众数在整个区间内出现的次数来决定该区间的众数。

实现

复杂度分析

  • 时间复杂度：O(n log n)。函数会求解2个长度为n/2的子问题，并做两边长度为n的线性扫描。因此分治算法的时间复杂度
    T(n) = 2T(n/2) + 2n
    根据主定理，主题满足第二种情况复杂度为Θ(nlogn)
  • 空间复杂度： O(log n)。 尽管分治法没有直接分配额外的数组空间，但在递归的过程中使用了额外的占空间。
    算法每次将数组从中间分成两部分，所以数组长度变为1之前需要进行O(log n)次递归。

Boyer-Moore投票算法

思路：
如果把众数记为 +1，其他数记为 -1，将他们全部加起来，显然和大于0，从结果本身我们可以看出众数比其他数多。

复杂度分析

  • 时间复杂度：O(n).
  • 空间复杂度：O(1).
