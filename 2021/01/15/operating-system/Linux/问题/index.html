调试串口问题

问题描述

Debian系统使用了两个USB转串口，一个用来Debug一个用来通信。但是发现调试串口登录不上。

处理过程

查看使用的调试串口

发现使用的串口错误，之后将对应的调试串口启用，另外一个关闭。

发现执行 systemctl start serial-getty@ttyUSB0.service 时，系统卡住了一段时间后显示 Dependency failed for Serial Getty on ttyUSB1 ,根据提示信息查找原因是因为没有找到 systemctl的对应服务配置，查看 /etc/systemd/system/getty.target.wants 下没有配置文件，从 /lib/systemd/system/ 下拷贝 serial-getty@.service 到 /etc/systemd/system/getty.target.wants 目录下，并改名字为 /serial-getty@ttyUSB0.service 。

然后执行重启端口命令，还是执行错误。在继续查找原因。

查看USB转串口，使用的驱动 lsmod | grep "serial" 为 ftdi_sio。 驱动是已加载的。加载驱动后会在 dev 目录下生成新的seweirial设备。 lsusb 可以看到 Ltd FT232 USB-Serail (UART) IC信息，应该有两个，现在只有一个。生成的设备节点在 /dev/serial/ 这个目录下也只能找到一个串口设备文件。所以是一个串口无法识别。

问题

ps -ef | grep healthBeat | grep -v grep | grep -v PPID | awk ‘{print $2} | xargs kill -9
这个命令在Debian-X86 系统上运行正常。
然而在raspberry系统上运行时会出错导致$?的值为1的错误返回值。在文件设置了’set -e’时不能继续执行。

改成如下内容后可以正常运行
‘’’ bash
pids=ps -ef | grep healthBeat | grep -v grep | grep -v PPID | awk ‘{print $2}
if [ ${ #ids} -eq 0 ]; then
echo $pids | xargs kill -9
fi
‘’’
