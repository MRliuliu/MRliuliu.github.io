gdb 使用

使用

1、编译代码
使用 gcc -ggdb3 编译之后，给生成的信息添加足够的调试信息。
2、使用 gdb 执行代码
开始执行。
进入gdb界面后使用如下操作：
r(run) 开始执行代码。
n(next) 从断点开始执行到协议个断点。
b(break) 给代码打断点。
l(list) 列出代码清单。
p(print) 打印变量
bt(backstrace) 打印调用栈

layout命令

1、显示源代码窗口
(gdb)layout src
2、显示寄存器窗口
(gdb)layout regs
3、显示汇编代码窗口
(gdb)layout asm
4、希纳是源代码和汇编代码
(gdb)layout split
5、显示窗口的大小
info win
6、刷新所有窗口
refresh 或者 <Ctrl + l>

设置断点

给系统调用设置断点

使用break指令加函数名设置断点。
b open
通过函数在文件行号添加断点
b main.c:12

查看断点
b 或者 info breakpoints

删除断点
delete 1

禁用断点
disable n

查看进程状态

查看现有的线程
info thread
然后使用thread <num>选择查看的线程

查看变量

使用p 命令加变量名查看 p a,如果是结构体或者字符串使用p info 看到的是地址，使用 p *info 可以查看整个数据的成员变量

查看数组变量：
p ArrayVar@ItemNum

查看局部变量
info local

GDB配合core文件

启用coredump文件输出，使用ulimit -a 命令查看core file size 大小是否为0，如果为0，使用 ulimit -c unlimited 取消大小限制。
然后运行程序，出现segment fault后再本地生成名为 core 的文件。

调试，使用gdb 程序名 core 就可以进入查看core文件的信息，进入gdb后，使用bt命令查看出错时的函数调用栈。
